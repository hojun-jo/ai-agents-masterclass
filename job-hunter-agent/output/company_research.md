# Company Overview
**Company:** Leaseweb  
**Industry:** Hosting & Cloud Services  
**Founded:** 1997  
**Employees:** ~600 globally  
**Headquarters:** Amsterdam, Netherlands (with offices and data centers across Europe, North America, and Asia)  
**Global Footprint:**  
- Operates multiple Tier‐III data centers and over 20 network Points of Presence (PoPs)  
- Serves enterprises, SaaS companies, MSPs and resellers  

Leaseweb provides dedicated servers, private & public cloud infrastructure, colocation, and CDN services. Their customers range from fast-growing startups to Fortune 500 enterprises.

## Role Context and Product Involvement
**Team:** Developer Platform (within Product Engineering)  
**Size:** ~90 engineers organized in small Scrum squads  
**Structure & Culture:**  
- **Flat Organization:** High autonomy, decentralized decision‐making  
- **Remote-First:** Core team meets twice per year onsite in Amsterdam  
- **End-to-End Ownership:** Each squad owns specific products or platforms  

**Product Focus:**  
- **Leaseweb API** – The unified “front door” for all Leaseweb services, powering websites, dashboards, CLI tools, and partner integrations  
- **Supporting Platforms:**  
  - Authentication & OAuth2 gateway  
  - Abuse & billing APIs  
  - Terraform provider (in beta)  
  - Internal SDKs & libraries (Go, PHP, Python)  
- **Tech Stack:** Golang (primary), PHP, Chef (infra as code), Elastic Stack (logging/metrics), Terraform, Kubernetes  

**Key Interfaces:**  
- Cross-functional with Product Managers, DevOps, Security, Customer Success  
- External: developer community (forums, GitHub), industry events, hackathons

## Likely Interview Topics
1. **Golang & API Design**  
   - Designing idiomatic, high-performance Go microservices  
   - RESTful vs. gRPC vs. OpenAPI specifications  
   - Error handling, context propagation, middleware patterns  

2. **Terraform Plugin Development**  
   - Terraform provider lifecycle & schema  
   - State management and resource CRUD operations  

3. **Authentication & Security**  
   - OAuth2 flows (Authorization Code, Client Credentials)  
   - JWT best practices, token revocation, rate limiting  

4. **System Architecture & Scalability**  
   - Horizontal scaling of stateless services  
   - Caching strategies (Redis, in-memory)  
   - Observability: metrics (Prometheus), logging (ELK), tracing (Jaeger)  

5. **Linux & DevOps Fundamentals**  
   - Containerization (Docker), orchestration (Kubernetes)  
   - Configuration management with Chef  
   - CI/CD pipelines, GitOps  

6. **Agile & Collaboration**  
   - Scrum ceremonies, sprint planning, backlog refinement  
   - Code reviews, pair programming, “you build it, you run it” mentality  
   - Working in a remote‐first, multicultural environment  

7. **Behavioral / Cultural Fit**  
   - Taking ownership in a flat structure  
   - Communicating complex technical topics to non-technical stakeholders  
   - Examples of past community engagement (blog posts, open-source contributions)

## Suggested Questions to Ask
1. **Strategic Vision & Roadmap**  
   - “How does the Leaseweb API roadmap align with the company’s 3-year strategic goals?”  
   - “What are the next big features or integrations planned for the Terraform provider?”  

2. **Team Structure & Collaboration**  
   - “Can you describe an end-to-end project cycle on the Developer Platform team?”  
   - “How do squads coordinate with Security and DevOps when rolling out critical API changes?”  

3. **Success Metrics & KPIs**  
   - “What key performance indicators do you use to measure API reliability and adoption?”  
   - “How do you balance velocity vs. stability in a customer-facing API?”  

4. **Technical Challenges & Pain Points**  
   - “What are the biggest technical obstacles the team has faced in scaling the API?”  
   - “Are there any legacy components or migration efforts underway in the platform?”  

5. **Professional Growth & Culture**

   - “How does Leaseweb support continuous learning—both technically (e.g., courses) and non-technically (e.g., leadership training)?”  

6. **Community & Hackathons**

7. **Work Model & Flexibility**  
   - “How does the hybrid policy work in practice—are there core days onsite, and how is time-zone alignment handled?”
